My Zola setup

I built this site using Zola. My goal was to upload my markdown files into a website of my own. One option was to use Obsidian and therefore Obsidian publish, an obsidian plugin to send my notes over the internet, but there was a big problem, I had to pay for it and as an iranian it had a lot of trouble that I didn't want to endure. So I searched for free solutions and Zola was the one that I chose.

What is Zola?

Zola (Static Site Generator) is a fast, modern static site generator written in Rust. It’s used to build websites from Markdown content and templates, then outputs static HTML files you can host anywhere. It's used for blogs, personal sites, documentation sites and etc. Zola sites can be hosted on GitHub Pages, Netlify, Vercel or any static hosting provider.

Features:

  • pretty fast
  • markdown based content
  • built in Sass/SCSS support
  • built in search
  • multilingual sites
  • easy to install with no runtime dependencies.

Installation

At first I installed Zola:

sudo pacman -S zola

I also had git installed.

sudo pacman -S git

Since I wanted to have a theme that was similar to Obsidian, I tried installing the Olivine theme. It had a quickstart repository:

git clone --depth 1 --recurse-submodules git@github.com:dongryul-kim/olivine-quickstart.git <your-folder-name>

Next, you enter the project:

cd my-zola-blog

In this folder you'll see these:

  • config.toml
  • content/
  • themes/olivine/

You need to change the base url in config.toml:

vim config.toml

or if you don't use vim:

nano config.toml

In this file, you'll see a line for base_url. Change it to your site's url. I am using github.io as the host for my website. ([How?])

base_url = "https://YOUR_USERNAME.github.io"
title = "My Blog"

Also make sure this exits:

theme = "olivine"
compile_sass = true

Save. Right now, your repo still points to dongryul-kim. First, check the repo:

git remote -v

You will see something like this:

origin git@github.com:dongryul-kim/olivine-quickstart.git

remove it:

git remote remove origin

Now test it locally:

zola serve

Open the https link and you'll see the blog with the theme. Since I wanted to connect the blog with my github repositories and share my notes via github, I tried to connect zola to a github repository so the notes will be displayed from this repo. To get there, you need to have 2 github repositories.

  1. Source repository (private or public) Name example: my-zola-blog This stores: Markdown notes Theme Templates Config files

  2. GitHub Pages repository (must be public) Name must be in this format: <your-username>.github.io This repository hosts the live website.
